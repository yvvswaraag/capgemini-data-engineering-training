# Day 6 – Data Pipeline Implementation

## Overview
On Day 6, I focused on building data pipelines using PySpark. I started with a simple pipeline and gradually enhanced it into a more structured and advanced version. The goal was to understand how real-world data engineering workflows operate, including data ingestion, transformation, and processing.

---

## What I Learned

### 1. Pipeline Concept
- Understood how a data pipeline works end-to-end  
- Learned how to organize multiple steps in a sequential flow  
- Realized the importance of modular and reusable code  

### 2. Data Ingestion
- Reading data from CSV and structured sources  
- Understanding schema inference  
- Selecting required columns  

### 3. Data Transformation
Applied:
- Filtering conditions  
- Column transformations  
- Case-When logic  
- Handling null values  

Learned how to chain multiple transformations effectively.

### 4. Mini Pipeline
Built a basic pipeline to:
- Load data  
- Clean data  
- Apply transformations  

This helped in understanding the step-by-step flow clearly.

### 5. Advanced Pipeline
- Extended the pipeline with multiple transformations  
- Improved structure and readability  
- Focused on writing scalable and clean code  

### 6. Final Pipeline Implementation
- Combined all concepts into a complete pipeline  
- Simulated a real-world data processing workflow  
- Improved logical thinking in pipeline design  

---

## Workflow
1. Load data from source  
2. Perform data cleaning  
3. Apply transformations  
4. Use conditional logic (case/when)  
5. Handle null values  
6. Generate final processed dataset  

---

## Challenges Faced
- Structuring pipeline steps properly  
- Managing multiple transformations  
- Ensuring correct data flow between steps  

---

## Outcome
- Gained understanding of real-world data pipelines  
- Improved confidence in PySpark transformations  
- Learned how to convert a simple script into a structured pipeline
