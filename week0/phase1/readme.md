In this phase, I began exploring PySpark and how it relates to traditional SQL operations.
I understood how data can be handled using DataFrames, which behave similarly to tables in SQL.

During this phase, I practiced:

Retrieving data using select()
Filtering records using filter()
Performing aggregations using groupBy()
Displaying outputs using show()

This helped me understand how data processing works in a distributed environment like PySpark.

#Understanding SQL vs PySpark

One of the key learnings was mapping SQL operations to PySpark functions:

SELECT → select()
WHERE → filter()
GROUP BY → groupBy()

This comparison made it easier to transition from SQL thinking to PySpark coding.
